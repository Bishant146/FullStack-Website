1. What is Full Stack Development?

Full stack development means one developer builds the entire website — both the side the user sees and the server logic running in the background. Before this module I always assumed those two parts were handled by different people, so I found it really interesting to learn that one person can be responsible for the whole thing.

It is split into two main sides:

  • Frontend — everything the user sees and interacts with on screen. Built using HTML, CSS and JavaScript.
  • Backend — the server and database. It stores data, handles logins and processes requests. Languages like Node.js or Python are used here.

A simple way to picture it is a login form. The form you fill in is the frontend. When you click submit, the backend checks your details against a database and sends back the right response. A full stack developer builds both of those parts.

Some common tools used across the full stack:

HTML CSS JavaScript Node.js MySQL Express.js MongoDB Git
Request flow — what happens when you log in
User (Browser) — fills in the login form and clicks submit
↕
Frontend (HTML + CSS + JS) — collects the data and sends it to the server
↕
Backend (Node.js) — validates the data and runs the business logic
↕
Database (MySQL) — stores and retrieves the user's information
